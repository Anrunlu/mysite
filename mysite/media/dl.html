润鲁笔记—计算机科学导论学习总结

历史上的计算机

  • 第一代计算机（1946-1958）

    • 体积大，运算速度低。

  • 第二代计算机（1958-1965）

    • 运算速度提升，体积减小，开始使用计算机算法语言。

  • 第三代计算机（1965-1970）

    • 以中小规模集成电路为电子器件，出现了操作系统。

  • 第四代计算机（1970年以后）

    • 大规模集成电路（LSI）和超大规模集成电路（VLSI），微处理器和微型计算机出现。

冯诺依曼体系结构

  • 冯·诺依曼（John von Neumann，1903年12月28日-1957年2月8日），美籍匈牙利数学家、计算机科学家、物理学家，是20世纪最重要的数学家之一。 [1-4] 冯·诺依曼是布达佩斯大学数学博士，在现代计算机、博弈论、核武器和生化武器等领域内的科学全才之一，被后人称为“现代计算机之父”、“博弈论之父”。

  • 讨论题：

    • 冯诺依曼体系结构包括几大部分，分别是什么？

      五大部分 分别为：运算器 控制器 存储器 输入设备和输出设备。

    • 冯诺依曼体系结构中控制器的功能是什么？

      发布命令的决策机构。

    • 冯诺依曼体系结构有几大特点，分别是什么？

      冯诺依曼结构采用单存储空间，程序指令和数据公用一个存储空间，使用 单一的数据和地址总线，取指令和取操作数都是通过一条总线分时进行的。并且当进行高速运算是，会造成数据传输通道的瓶颈现象，其工作速度较慢。

    def fa(a, b ,c):
        """全加器"""
        Carry = (a and b) or (a and c) or (b and c)
        Sum = (a and b and c) or (a and (not b) and (not c)) or \
            ((not a) and b and (not c)) or ((not a) and (not b) and c)
        return Carry, Sum

    def add(x, y):
        """多位相加"""
        while len(x) < len(y): x = [False] + x
        while len(y) < len(x): y = [False] + y

        Carry = False
        for i in range(len(x) - 1, -1, -1):
            Carry, Sum = fa(x[i], y[i], Carry)
            L = [Sum] + L
        return(Carry, L)

汇编指令的概念

  • 基本汇编指令

    • load 指令

      load R1, (address)

    • mov 指令

      • mov R1, constant

        constant 为16进制常数

      • mov R2, R1

        将 R1 中的值赋值给 R2

    • add 指令

      • add R2, R1, constant
      • add R1, R1, R2

    • sub 指令

      和 add 一样，只不过是减法

    • shiftl 指令

      左移位指令

      shiftl R3, R1, 05h

      代表 R1 的二进制数左移5位，移出那5位填0，再将最终值填入 R3， 05h 也可以用寄存器表示。

      R1 左移一位就相当于乘以 2，左移两位就相当于乘以 4

    • shiftr 指令

      向右移位，移完后那些最高位填 0

    • store 指令

      store (address), R1

    • slt 指令（）

      • slt R4, R1, R2

        如果 R1 小于 R2 ，则 R4 置 1， 否则置 0

      • slt R4, R1, constant

    • sle 指令

      • sle R4, R1, R2

        如果 R1 小于或等于 R2 ，则 R4 置 1， 否则置 0

      • sle R4, R1, constant

    • beqz 指令（Branch If Equals Zero）

      beqz R4, label2

      如果 R4 = 0 ，则跳转到标签 label2 标记的指令块处

    • goto 指令

      goto label3

      表示跳转到标签 label3 标记的指令处执行

  • CPU执行程序语句 a = 20 时，基本的操作是什么？

    • mov R1, 14h
    • store(1000), R1

  • 用 shiftl 和 add 指令将 R1 乘以 0ch（十进制的12），存入 R3。add指令越少越好。

    • mov R2, R1
    • add R1, R1, R2
    • add R1, R1, R2
    • shiftl R3, R1, 02h

中断

  • 硬件中断

    • 由硬件发出的中断，包括 I/O 设备发出的数据交换请求、时钟中断等。

  • 软件中断

    • 由应用程序触发的中断。

      • 内核态与用户态

        CPU 指令分为需要特权的指令（Privileged）和一般的指令（Non-Privileged）。

        所有的 I/O 指令都属于需要特权的指令。必须是操作系统才能执行，一般用户不能执行。

        CPU 中的状态寄存器中显示当前的CPU处于内核态还是用户态。

        使用软件中断的方式让 CPU 从用户态转成内核态。

        最底层的就是执行 int 指令（intel x86下， ARM 下是 swi 指令），执行这条指令时，CPU 会自动的将状态置为内核态。

        • 讨论1：能不能有一个 switch_to_kernel_mode 的指令，目的是将状态变成内核态。有了这样的指令，会有什么问题？

          不能。这样做用户能绕过操作系统转换到内核态，操作系统就失去了对硬件的保护。

        • 讨论2：为什么利用这种内核态和用户态的保护方式，普通用户程序不能利用 int 进入内核态后再胡作非为呢？

          利用这样的保护方式，普通用户程序进入内核态后所做的所有操作都是特定的中断服务程序（内核程序），用户程序无法进行其他非法的操作。
